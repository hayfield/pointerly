Class Detail
Pointerly.Setup.Environment()
A description of the object properties which can be used to setup a Pointerly.Environment and are passed to Pointerly.Init.
The properties can be on a regular object, this merely describes what they are.
Field Detail
{boolean} autoGenerateShapes
Indicates whether shapes should be generated immediately when the environment is created.
{string} clickType
Defines the type of click handler to use.
If undefined, clicking will have no effect.
The possible values available by default are mouse and screencenter. Custom click handlers may be defined.
{boolean} displayMouseTrail
Indicates whether a mouse trail should be shown all the time, even when not replaying.
{boolean} fixedViewBetweenEvents
Indicates whether the view of the canvas should remain constant without explicit update calls.
If set to anything that evaluates to true, the view will remain fixed, otherwise the view will update on a regular schedule.
{Pointerly.Setup.HomePosition|object} homePosition
Defines the setup for the Home Position.
If undefined, there will be no Home Position.
other
Other properties may be defined.
Some of these additional properties may be required to customise how shape generators or click handlers work.
{boolean} replaying
Indicates whether the environment is used for replaying logged data.
If setting up a replay environment, this should be set to true. All other properties will then be ignored.
{string|function} shapeLayout
Defines the method in which to layout shapes.
If it's a function, that function will be used to layout the shapes.
If it's a string, the registered method of shape generation with that name will be used. Custom methods of named shape generation may be defined.
